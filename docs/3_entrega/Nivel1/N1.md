# C4 Nível 1 - Diagrama de Contexto do Sistema

## 1. Introdução
O Diagrama de Contexto (Nível 1) apresenta uma visão de alto nível do **Sistema da Clínica Hospitalar**. O objetivo deste diagrama é definir as fronteiras do sistema, identificando quem interage com ele (atores humanos) e quais as principais responsabilidades funcionais da plataforma, abstraindo-se de detalhes técnicos ou de implementação.

Nesta visão, o sistema é tratado como uma "caixa negra", focando-se no valor que entrega aos seus utilizadores.

## 2. Atores e Papéis
O sistema interage com quatro tipos distintos de utilizadores, cada um com necessidades específicas:

*   **Utilizador Anónimo:** Representa qualquer visitante que ainda não possui credenciais no sistema. A sua única interação permitida é o processo de *onboarding* (registo), que inicia o fluxo de criação de identidade e perfil de paciente.
*   **Paciente:** Um utilizador registado e autenticado. Interage com o sistema para gerir a sua saúde, nomeadamente através do agendamento de consultas, pesquisa de médicos disponíveis e consulta do seu histórico.
*   **Médico:** O profissional de saúde que utiliza a plataforma como ferramenta de trabalho. O seu foco principal é a componente clínica, sendo responsável por registar detalhes das consultas (diagnósticos, receitas, notas) no prontuário eletrónico.
*   **Administrador:** O gestor operacional da clínica. Possui privilégios elevados para gerir os dados mestres (perfis de médicos e pacientes), supervisionar o funcionamento do sistema e visualizar relatórios estatísticos (ex: métricas de duração de consultas).

## 3. Sistema Central
*   **Sistema da Clínica Hospitalar:** É o sistema de software desenvolvido no âmbito do projeto. Engloba toda a lógica de negócio, incluindo a gestão de identidades, o motor de agendamentos (com transações distribuídas), a gestão de registos clínicos e a persistência de dados.

## 4. Principais Interações
As relações ilustradas no diagrama descrevem os fluxos de valor principais:

1.  **Registo (Anonymous -> System):** O ponto de entrada para novos utentes.
2.  **Gestão de Consultas (Patient -> System):** O paciente inicia processos de agendamento (escrita) e consulta informação (leitura).
3.  **Registo Clínico (Physician -> System):** O médico alimenta o sistema com dados clínicos resultantes das consultas realizadas.
4.  **Gestão e Análise (Admin -> System):** O administrador configura o sistema e consome dados agregados para tomada de decisão.

## 5. Enquadramento no Assignment 3
Embora este diagrama apresente uma visão simplificada, o interior do **Sistema da Clínica Hospitalar** é composto por uma arquitetura complexa de microsserviços distribuídos. As interações aqui representadas como linhas simples (ex: "Gere as suas consultas") desencadeiam internamente padrões avançados como **Sagas Coreografadas** para garantir a consistência entre agendamentos e disponibilidade médica, e **CQRS** para separar as operações de leitura das de escrita.
